# Overview

인천영어마을의 기존 홈페이지는 정보 배치가 중구난방이고 색상이 지나치게 다양하여, 주 사용자층인 학부모와 아이들이 원하는 정보를 찾는 데 큰 혼란을 겪는 구조였습니다. 특히 전반적인 UI 구조가 체계적이지 못해 사용자의 시선이 사방으로 분산되었고, 사이트가 전달하고자 하는 핵심 메시지를 파악하기 어렵다는 문제가 있었습니다.

그래서 사용자층을 명확하게 정의하여 UI를 직관적으로 개선하고, 복잡한 정보들이 사용자들에게 효율적으로 전달되도록 구조를 재설계하는 것을 목표로 프로젝트를 진행했습니다.


# Problem & Solution

## Problem

실제로 사이트를 이용해 보았을 때, 지나치게 알록달록한 색상과 기준 없이 배치된 요소들 때문에 시선이 사방으로 분산되어 원하는 정보를 찾는데 어려움을 겪었습니다. 

이러한 비논리적인 UI와 정보 배치의 불균형이 사용자에게 높은 시각적 피로감을 주고 있어 개선이 필요하다고 생각했습니다.

## Solution

사용자층을 학부모와 아이들로 명확하게 정의하고, 이들에게 신뢰감과 친근함을 줄 수 있는 사용자 친화적인 디자인으로 개선했습니다. 

특히 사이트의 정체성을 보여주는 마스코트 캐릭터를 적극적으로 활용하여 시각적 긴장감을 낮추고, 사이트 방문 목적에 맞게 교육 프로그램 신청 및 안내 콘텐츠를 메인 상단에 배치하여 정보의 접근성을 높였습니다. 

또한 메인 색상을 지정하고 SCSS를 활용해 일관된 컬러 가이드를 적용함으로써 시선 분산을 막고 시각적 안정감을 주는 방향으로 설계했습니다.

## Result

결과적으로 마스코트와 정제된 UI가 조화를 이루며 사이트의 정체성이 훨씬 뚜렷해지고, 핵심 서비스에 대한 접근성이 매우 좋아졌습니다. 

또한 주 사용자층의 눈높이에 맞춘 직관적인 구조 덕분에 시각적 피로도도 낮아지게 되었습니다. 

더불어 다양한 기기에서의 반응형 환경을 구축했기 때문에, 모바일 환경에 익숙한 젊은 세대 학부모들도 언제 어디서나 편하게 사이트를 이용할 수 있도록 사용성이 개선되었습니다.


# Main Feature

- 어린이와 학부모를 고려한 캐릭터 기반 메인 UI 구성
- 공지사항 및 프로그램 정보 구조화
- 달력 기반 프로그램 예약 시스템 구현
- 학습 테스트를 통한 레벨 분류 기능 구현
- 지도 연동을 통한 위치 안내 기능 제공
- 반응형 레이아웃 적용


# What I Learned

사용자 타겟에 맞춘 디자인 방향성이 서비스의 정체성에 얼마나 중요한지 체감했습니다. 특히 마스코트와 컬러 가이드를 활용해 사용자에게 친근함과 시각적인 편안함을 주는 과정의 가치를 깊이 느낄 수 있었습니다.

또한 2인 협업을 진행하며 서로 다른 의견을 조율하고 각기 다른 코드 방식을 하나로 취합하는 과정이 쉽지 않았습니다. 하지만 이 과정에서 사람마다 구조를 설계하는 방식이 다를 수 있다는 점을 경험했습니다.

코드를 비교해보면서, 팀원은 페이지의 시각적 구조를 기준으로 HTML을 구성하고 반복되는 UI를 개별적으로 분리하기보다 하나의 구조 안에서 순차적으로 배치하고 스타일을 정의하는 방식을 사용했습니다. 반면 저는 기능과 동작을 기준으로 구조를 나누어 사용자 입력, 문제 출력, 페이지네이션, 결과 영역 등을 각각 분리해 설계했습니다.

팀원의 방식은 전체 화면 구조를 한눈에 파악하고 디자인 흐름을 따라가기 쉽다는 장점이 있었지만, 특정 요소를 수정할 경우 영향 범위가 넓어질 수 있다는 단점이 있었습니다.

반대로 저의 방식은 기능 수정이나 확장 시 영향을 받는 범위를 줄일 수 있어 유지보수 측면에서 효율적이었지만, 초기 구조를 나누는 과정에서 설계에 더 많은 시간이 필요하다는 점이 있었습니다.

이러한 경험을 통해 화면 중심 구조와 기능 중심 구조 각각의 특징을 비교할 수 있었고, 상황에 따라 적절한 방식을 선택하는 것이 중요하다는 점을 인식하게 되었습니다.
