# Overview

기존 포트폴리오는 HTML, CSS, JavaScript로 구현되어 있었으며, 정적인 구조로 인해 콘텐츠 확장과 유지보수에 한계가 있었습니다.

이에 따라 컴포넌트 기반 구조와 데이터 중심 렌더링을 적용하기 위해 React와 TypeScript를 도입하여 전체 구조를 재설계하는 것을 목표로 프로젝트를 진행했습니다.


# Problem & Solution

## Problem

기존 구조는 페이지와 UI 요소가 명확히 분리되어 있지 않아 코드 재사용이 어렵고, 기능이 추가될수록 구조가 복잡해지는 문제가 있었습니다.

또한 프로젝트 데이터가 코드에 직접 하드코딩되어 있어, 콘텐츠를 수정하거나 확장할 때 유지보수 비용이 증가하는 한계가 있었습니다.

## Solution

React를 기반으로 UI를 컴포넌트 단위로 분리하고, 각 영역을 역할에 맞게 구조화하여 재사용 가능한 형태로 설계했습니다.

또한 프로젝트 정보를 md 파일과 데이터 구조로 분리하여 관리함으로써, UI와 콘텐츠를 독립적으로 유지할 수 있는 구조로 개선했습니다.

TypeScript를 도입해 컴포넌트 간 데이터 전달 시 타입을 명확히 정의하고, 데이터 흐름을 보다 안정적으로 관리할 수 있도록 설계했습니다.

## Result

컴포넌트 기반 구조로 전환함에 따라 UI 재사용성이 향상되었고, 새로운 프로젝트를 추가하거나 수정하는 과정이 훨씬 단순해졌습니다.

또한 데이터와 UI를 분리한 구조를 통해 유지보수성이 개선되었으며, 전체 코드의 가독성과 확장성이 향상되었습니다.

TypeScript 적용을 통해 예상치 못한 오류를 사전에 방지할 수 있는 기반을 마련하게 되었습니다.


# Main Feature

- React 기반 컴포넌트 구조 설계
- TypeScript를 활용한 타입 정의 및 상태 관리
- md 기반 프로젝트 데이터 관리 구조 구현
- 프로젝트 필터링 및 상세 페이지 구현
- 반응형 UI 및 다크 모드 적용


# What I Learned

정적인 구조에서 컴포넌트 기반 구조로 전환하는 과정에서, UI를 단순히 구현하는 것이 아니라 재사용성과 확장성을 고려한 설계가 중요하다는 점을 체감했습니다.

특히 데이터와 UI를 분리하는 구조를 직접 설계하며, 콘텐츠가 변경되더라도 코드 수정 범위를 최소화할 수 있는 구조의 필요성을 인식하게 되었습니다.

또한 TypeScript를 적용하는 과정에서 타입 정의가 단순한 문법이 아니라, 데이터 흐름을 명확하게 하고 코드의 안정성을 높이는 중요한 역할을 한다는 점을 경험할 수 있었습니다.
