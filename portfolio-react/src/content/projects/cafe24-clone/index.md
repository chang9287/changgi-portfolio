# Overview

React 기반의 컴포넌트 설계와 상태 관리 구조에 대한 이해를 높이고, 로그인부터 상품 탐색, 상세 조회, 장바구니까지 이어지는 실제 쇼핑몰 서비스의 사용자 흐름을 직접 구현하며 각 페이지 간 데이터 전달과 연결 구조에 집중했습니다.

특히 여러 페이지에서 공통으로 사용되는 데이터를 효율적으로 관리하고, 컴포넌트 재사용성을 고려한 구조를 설계하는 것을 목표로 프로젝트를 진행했습니다.


# Problem & Solution

## Problem

장바구니 기능을 구현하는 과정에서, 여러 페이지에서 동일한 데이터를 어떻게 공유하고 유지해야 하는지에 대한 이해가 부족했습니다.

특히 상품 목록에서 담은 데이터가 장바구니 페이지까지 어떻게 전달되고 상태로 유지되는지 구조를 명확히 이해하지 못해 구현에 어려움을 겪었습니다.

## Solution

useContext를 활용해 App.js에서 Context Provider로 감싸 여러 페이지에서 동일한 상태를 공유할 수 있도록 구성하고, 각 컴포넌트에서는 Context를 통해 필요한 데이터를 직접 참조하도록 구현했습니다.

이를 통해 장바구니 데이터를 전역 상태로 관리하는 구조를 설계했습니다.

## Result

전역 상태 관리 개념을 직접 코드로 구현해보며 리액트 특유의 데이터 흐름을 명확히 이해하게 되었습니다.

이전에는 부모-자식 간의 수직적인 전달 방식에만 익숙했지만, useContext를 활용해 여러 컴포넌트가 하나의 상태를 함께 공유할 수 있다는 점을 알게 되었고, 데이터가 여러 컴포넌트 사이에서 어떻게 흐르고 유지되는지에 대한 흐름을 이해하게 되었습니다.


# Main Feature

- Swiper를 활용한 메인 슬라이더 구현
- 카테고리별, 베스트, 신상품 필터링 기능
- 재사용 가능한 컴포넌트 설계
- CartContext를 활용한 전역 상태 관리
- 장바구니 담기, 수량 조절, 자동 금액 계산 기능

- 할인율 및 무료배송 기준 자동 계산 로직


# What I Learned

처음 작업을 할 때는 전역 상태 관리에 대한 개념이 부족한 상태였습니다.

장바구니 데이터도 부모에서 자식으로 props를 통해 전달하며 처리할 수 있을 것이라고 생각했습니다. 하지만 데이터를 주고받는 방식이 1:1 구조로 이루어지다 보니, 실제로는 여러 페이지에서 장바구니로 데이터를 전달해야 하는 상황에서 한계를 느끼게 되었습니다.

예를 들어 메인 페이지의 장바구니 버튼, 상품의 상세 페이지에서 수량 선택 후 담기 기능 등 데이터를 보내는 위치는 여러 곳이지만, 이를 props 방식으로만 전달하려고 하면 구조가 복잡해지고 관리가 어렵다는 점을 깨닫게 되었습니다.

이런 고민을 해결하는 과정에서, 여러 페이지에서 동일한 장바구니 데이터를 함께 사용하고 항상 같은 상태를 유지 하기 위해 전역 상태 관리가 필요하다는 것을 알게 되었습니다.

또한 useContext를 통해 상태를 상위에서 한 번만 정의하면, 필요한 컴포넌트에서 별도의 props 전달 없이 직접 데이터를 가져와 사용할 수 있다는 점을 이해하게 되었고, 이를 통해 여러 컴포넌트가 하나의 상태를 공유하는 구조를 구현할 수 있었습니다.
